# Billing: Billing Period Tables

This is a proposed database shape for the billing period POC. It supports:

- admin-managed billing periods
- lifecycle statuses: draft, published, archived
- mapping a billing period to an actual academic year and term
- legacy billing-period fields from the current billing setup screen
- a future place to record billing runs when the `Run Billing` action becomes real

Implemented by Flyway migration:
`api/src/main/resources/db/migration/V30__create_billing_period_tables.sql`.

## Tables

### `billing_period`

Stores one billing period setup record.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `billing_period_id` | `BIGINT` | Yes | Identity primary key |
| `name` | `VARCHAR(64)` | Yes | Admin-facing period name, unique, example: `2026SP` |
| `description` | `VARCHAR(255)` | Yes | Display description, example: `Spring 2026` |
| `type` | `VARCHAR(32)` | Yes | Example values: `STANDARD`, `OPEN_ENDED` |
| `status` | `VARCHAR(32)` | Yes | `DRAFT`, `PUBLISHED`, or `ARCHIVED` |
| `academic_year_id` | `BIGINT` | No | Actual academic year mapping |
| `term_id` | `BIGINT` | No | Actual academic term mapping |
| `start_date` | `DATE` | No | Billing period start date |
| `end_date` | `DATE` | No | Billing period end date |
| `actual_start_preliminary_end_date` | `DATE` | No | Legacy actual start/preliminary end date |
| `tax_academic_year` | `VARCHAR(32)` | No | 1098-T academic year value from legacy UI |
| `tax_academic_year_label` | `VARCHAR(64)` | No | 1098-T year label, example: `2025-2026` |
| `tax_academic_term_code` | `VARCHAR(32)` | No | 1098-T term code, example: `SP` |
| `tax_academic_term_name` | `VARCHAR(128)` | No | 1098-T term name, example: `Spring` |
| `financial_aid_period_code` | `VARCHAR(64)` | No | Financial aid period code |
| `financial_aid_period_name` | `VARCHAR(128)` | No | Financial aid period name |
| `course_billing_basis` | `VARCHAR(64)` | Yes | Date basis for course billing |
| `non_course_billing_basis` | `VARCHAR(64)` | Yes | Date basis for non-course billing |
| `actual_from_to_days` | `INTEGER` | Yes | Offset days for actual billing |
| `preliminary_from_to_days` | `INTEGER` | Yes | Offset days for preliminary billing |
| `active` | `BOOLEAN` | Yes | Legacy active checkbox, separate from lifecycle status |
| `allow_re_billing` | `BOOLEAN` | Yes | Legacy RE billing flag |
| `allow_ar_billing` | `BOOLEAN` | Yes | Legacy AR billing flag |
| `include_in_ar_statements` | `BOOLEAN` | Yes | Legacy AR statement flag |
| `allow_billing_in_campus_portal` | `BOOLEAN` | Yes | Legacy campus portal billing flag |
| `run_prelim_in_campus_portal_only` | `BOOLEAN` | Yes | Legacy prelim-only flag |
| `academic_records_mapped` | `BOOLEAN` | Yes | Legacy mapping indicator |
| `children_assigned` | `BOOLEAN` | Yes | Legacy child-assignment indicator |
| `created_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP`, maintained by trigger |
| `created_by_user_id` | `BIGINT` | No | Admin who created the period |
| `updated_by_user_id` | `BIGINT` | No | Last admin to edit the period |

### `billing_period_run`

Future table for the `Run Billing` action. This lets us keep billing-period setup separate from each execution attempt.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `billing_period_run_id` | `BIGINT` | Yes | Identity primary key |
| `billing_period_id` | `BIGINT` | Yes | Billing period being run |
| `status` | `VARCHAR(32)` | Yes | Example values: `QUEUED`, `RUNNING`, `COMPLETED`, `FAILED` |
| `billing_period_status_at_run` | `VARCHAR(32)` | Yes | Billing period lifecycle status when the run was created |
| `started_at` | `TIMESTAMP` | No | When execution started |
| `completed_at` | `TIMESTAMP` | No | When execution finished |
| `trigger_source` | `VARCHAR(32)` | Yes | `USER` or `SYSTEM` |
| `triggered_by_user_id` | `BIGINT` | No | User who triggered the run; null for system runs |
| `message` | `TEXT` | No | Human-readable result or failure detail |
| `created_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP`, maintained by trigger |
| `created_by_user_id` | `BIGINT` | No | Admin/system user that created the run row |
| `updated_by_user_id` | `BIGINT` | No | Last admin/system user to update the run |

## SQL Draft

```sql
CREATE TABLE billing_period (
    billing_period_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name VARCHAR(64) NOT NULL,
    description VARCHAR(255) NOT NULL,
    type VARCHAR(32) NOT NULL DEFAULT 'STANDARD',
    status VARCHAR(32) NOT NULL DEFAULT 'DRAFT',
    academic_year_id BIGINT NULL,
    term_id BIGINT NULL,
    start_date DATE NULL,
    end_date DATE NULL,
    actual_start_preliminary_end_date DATE NULL,
    tax_academic_year VARCHAR(32) NULL,
    tax_academic_year_label VARCHAR(64) NULL,
    tax_academic_term_code VARCHAR(32) NULL,
    tax_academic_term_name VARCHAR(128) NULL,
    financial_aid_period_code VARCHAR(64) NULL,
    financial_aid_period_name VARCHAR(128) NULL,
    course_billing_basis VARCHAR(64) NOT NULL DEFAULT 'BILLING_PERIOD_START_DATE',
    non_course_billing_basis VARCHAR(64) NOT NULL DEFAULT 'BILLING_PERIOD_START_DATE',
    actual_from_to_days INTEGER NOT NULL DEFAULT 0,
    preliminary_from_to_days INTEGER NOT NULL DEFAULT 0,
    active BOOLEAN NOT NULL DEFAULT TRUE,
    allow_re_billing BOOLEAN NOT NULL DEFAULT FALSE,
    allow_ar_billing BOOLEAN NOT NULL DEFAULT TRUE,
    include_in_ar_statements BOOLEAN NOT NULL DEFAULT FALSE,
    allow_billing_in_campus_portal BOOLEAN NOT NULL DEFAULT FALSE,
    run_prelim_in_campus_portal_only BOOLEAN NOT NULL DEFAULT FALSE,
    academic_records_mapped BOOLEAN NOT NULL DEFAULT FALSE,
    children_assigned BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by_user_id BIGINT NULL,
    updated_by_user_id BIGINT NULL,

    CONSTRAINT uq_billing_period_name
        UNIQUE (name),

    CONSTRAINT chk_billing_period_name_not_blank
        CHECK (LENGTH(TRIM(name)) > 0),

    CONSTRAINT chk_billing_period_description_not_blank
        CHECK (LENGTH(TRIM(description)) > 0),

    CONSTRAINT chk_billing_period_type
        CHECK (type IN ('STANDARD', 'OPEN_ENDED')),

    CONSTRAINT chk_billing_period_status
        CHECK (status IN ('DRAFT', 'PUBLISHED', 'ARCHIVED')),

    CONSTRAINT chk_billing_period_course_billing_basis
        CHECK (course_billing_basis IN (
            'BILLING_PERIOD_START_DATE',
            'BILLING_PERIOD_END_DATE',
            'ACTUAL_START_PRELIMINARY_END_DATE'
        )),

    CONSTRAINT chk_billing_period_non_course_billing_basis
        CHECK (non_course_billing_basis IN (
            'BILLING_PERIOD_START_DATE',
            'BILLING_PERIOD_END_DATE',
            'ACTUAL_START_PRELIMINARY_END_DATE'
        )),

    CONSTRAINT chk_billing_period_date_range
        CHECK (start_date IS NULL OR end_date IS NULL OR start_date <= end_date),

    CONSTRAINT fk_billing_period_academic_year
        FOREIGN KEY (academic_year_id)
            REFERENCES academic_year(academic_year_id),

    CONSTRAINT fk_billing_period_term
        FOREIGN KEY (term_id)
            REFERENCES academic_term(term_id),

    CONSTRAINT fk_billing_period_created_by
        FOREIGN KEY (created_by_user_id)
            REFERENCES users(id),

    CONSTRAINT fk_billing_period_updated_by
        FOREIGN KEY (updated_by_user_id)
            REFERENCES users(id)
);

CREATE TABLE billing_period_run (
    billing_period_run_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    billing_period_id BIGINT NOT NULL,
    status VARCHAR(32) NOT NULL DEFAULT 'QUEUED',
    billing_period_status_at_run VARCHAR(32) NOT NULL,
    started_at TIMESTAMP NULL,
    completed_at TIMESTAMP NULL,
    trigger_source VARCHAR(32) NOT NULL DEFAULT 'USER',
    triggered_by_user_id BIGINT NULL,
    message TEXT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by_user_id BIGINT NULL,
    updated_by_user_id BIGINT NULL,

    CONSTRAINT chk_billing_period_run_status
        CHECK (status IN ('QUEUED', 'RUNNING', 'COMPLETED', 'FAILED')),

    CONSTRAINT chk_billing_period_run_period_status_at_run
        CHECK (billing_period_status_at_run IN ('DRAFT', 'PUBLISHED', 'ARCHIVED')),

    CONSTRAINT chk_billing_period_run_completed_after_started
        CHECK (started_at IS NULL OR completed_at IS NULL OR started_at <= completed_at),

    CONSTRAINT chk_billing_period_run_trigger_source
        CHECK (trigger_source IN ('USER', 'SYSTEM')),

    CONSTRAINT chk_billing_period_run_triggered_by_user
        CHECK (
            (trigger_source = 'USER' AND triggered_by_user_id IS NOT NULL)
            OR
            (trigger_source = 'SYSTEM' AND triggered_by_user_id IS NULL)
        ),

    CONSTRAINT fk_billing_period_run_billing_period
        FOREIGN KEY (billing_period_id)
            REFERENCES billing_period(billing_period_id),

    CONSTRAINT fk_billing_period_run_triggered_by_user
        FOREIGN KEY (triggered_by_user_id)
            REFERENCES users(id),

    CONSTRAINT fk_billing_period_run_created_by
        FOREIGN KEY (created_by_user_id)
            REFERENCES users(id),

    CONSTRAINT fk_billing_period_run_updated_by
        FOREIGN KEY (updated_by_user_id)
            REFERENCES users(id)
);

CREATE INDEX idx_billing_period_academic_year_term
    ON billing_period (academic_year_id, term_id);

CREATE INDEX idx_billing_period_status
    ON billing_period (status);

CREATE INDEX idx_billing_period_created_by
    ON billing_period (created_by_user_id);

CREATE INDEX idx_billing_period_updated_by
    ON billing_period (updated_by_user_id);

CREATE INDEX idx_billing_period_run_billing_period
    ON billing_period_run (billing_period_id);

CREATE INDEX idx_billing_period_run_status
    ON billing_period_run (status);

CREATE INDEX idx_billing_period_run_period_status_at_run
    ON billing_period_run (billing_period_status_at_run);

CREATE INDEX idx_billing_period_run_triggered_by_user
    ON billing_period_run (triggered_by_user_id);

CREATE INDEX idx_billing_period_run_created_by
    ON billing_period_run (created_by_user_id);

CREATE INDEX idx_billing_period_run_updated_by
    ON billing_period_run (updated_by_user_id);

CREATE TRIGGER trg_billing_period_updated_at
    BEFORE UPDATE ON billing_period
    FOR EACH ROW
    EXECUTE FUNCTION set_updated_at_timestamp();

CREATE TRIGGER trg_billing_period_run_updated_at
    BEFORE UPDATE ON billing_period_run
    FOR EACH ROW
    EXECUTE FUNCTION set_updated_at_timestamp();
```

## Notes

- `status` is the lifecycle stepper value. It is separate from the legacy `active` checkbox.
- `academic_year_id` and `term_id` are nullable so a draft can exist before mapping is complete.
- `billing_period_run` is intentionally minimal until we know what the billing calculation produces.
- `billing_period_status_at_run` is a snapshot. It should not change if the billing period status changes later.
- `trigger_source` plus `triggered_by_user_id` lets the UI display either a person or a system-triggered run.
- The legacy flags may eventually become derived values instead of stored values. They are included here because they exist in the current screen and frontend stub.
