# Billing: Tuition Code Tables

This is a proposed database shape for the first billing slice. It supports:

- admin-managed tuition codes with a code and name
- one manually assigned tuition code per student
- no student-facing exposure of tuition code data
- normal SIS audit columns and `updated_at` trigger behavior

Implemented by Flyway migration:
`api/src/main/resources/db/migration/V29__create_billing_tuition_code_tables.sql`.

## Tables

### `tuition_code`

Stores the managed list of tuition codes.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `tuition_code_id` | `BIGINT` | Yes | Identity primary key |
| `code` | `VARCHAR(32)` | Yes | Short admin-facing code, unique |
| `name` | `VARCHAR(255)` | Yes | Display name |
| `created_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP`, maintained by trigger |
| `created_by_user_id` | `BIGINT` | No | Admin who created the code |
| `updated_by_user_id` | `BIGINT` | No | Last admin to edit the code |

### `student_tuition_code_assignment`

Stores the current manual tuition-code assignment for a student. Absence of a row means the student has no assigned tuition code.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `student_tuition_code_assignment_id` | `BIGINT` | Yes | Identity primary key |
| `student_id` | `BIGINT` | Yes | Student receiving the tuition code |
| `tuition_code_id` | `BIGINT` | Yes | Assigned tuition code |
| `created_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP` |
| `updated_at` | `TIMESTAMP` | Yes | Defaults to `CURRENT_TIMESTAMP`, maintained by trigger |
| `created_by_user_id` | `BIGINT` | No | Admin who created the assignment |
| `updated_by_user_id` | `BIGINT` | No | Last admin to update the assignment |

## SQL

```sql
CREATE TABLE tuition_code (
    tuition_code_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    code VARCHAR(32) NOT NULL,
    name VARCHAR(255) NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by_user_id BIGINT NULL,
    updated_by_user_id BIGINT NULL,

    CONSTRAINT uq_tuition_code_code
        UNIQUE (code),

    CONSTRAINT chk_tuition_code_code_not_blank
        CHECK (LENGTH(TRIM(code)) > 0),

    CONSTRAINT chk_tuition_code_name_not_blank
        CHECK (LENGTH(TRIM(name)) > 0),

    CONSTRAINT fk_tuition_code_created_by
        FOREIGN KEY (created_by_user_id)
            REFERENCES users(id),

    CONSTRAINT fk_tuition_code_updated_by
        FOREIGN KEY (updated_by_user_id)
            REFERENCES users(id)
);

CREATE TABLE student_tuition_code_assignment (
    student_tuition_code_assignment_id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    student_id BIGINT NOT NULL,
    tuition_code_id BIGINT NOT NULL,
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by_user_id BIGINT NULL,
    updated_by_user_id BIGINT NULL,

    CONSTRAINT uq_student_tuition_code_assignment_student
        UNIQUE (student_id),

    CONSTRAINT fk_student_tuition_code_assignment_student
        FOREIGN KEY (student_id)
            REFERENCES student(student_id) ON DELETE CASCADE,

    CONSTRAINT fk_student_tuition_code_assignment_tuition_code
        FOREIGN KEY (tuition_code_id)
            REFERENCES tuition_code(tuition_code_id),

    CONSTRAINT fk_student_tuition_code_assignment_created_by
        FOREIGN KEY (created_by_user_id)
            REFERENCES users(id),

    CONSTRAINT fk_student_tuition_code_assignment_updated_by
        FOREIGN KEY (updated_by_user_id)
            REFERENCES users(id)
);

CREATE INDEX idx_tuition_code_updated_by
    ON tuition_code (updated_by_user_id);

CREATE INDEX idx_tuition_code_created_by
    ON tuition_code (created_by_user_id);

CREATE INDEX idx_student_tuition_code_assignment_tuition_code
    ON student_tuition_code_assignment (tuition_code_id);

CREATE INDEX idx_student_tuition_code_assignment_created_by
    ON student_tuition_code_assignment (created_by_user_id);

CREATE INDEX idx_student_tuition_code_assignment_updated_by
    ON student_tuition_code_assignment (updated_by_user_id);

CREATE TRIGGER trg_tuition_code_updated_at
    BEFORE UPDATE ON tuition_code
    FOR EACH ROW
    EXECUTE FUNCTION set_updated_at_timestamp();

CREATE TRIGGER trg_student_tuition_code_assignment_updated_at
    BEFORE UPDATE ON student_tuition_code_assignment
    FOR EACH ROW
    EXECUTE FUNCTION set_updated_at_timestamp();
```

## Notes

- Keep `tuition_code.code` uppercase in application validation. The database only requires non-blank uniqueness.
- `student_tuition_code_assignment` is deliberately current-state only. If billing needs history later, add an effective-dated table or assignment history table instead of overloading this first simple table.
- Student visibility should be enforced in routes/services. These tables can exist in the same schema without making tuition codes available to student roles.
