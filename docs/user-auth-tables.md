# User Auth Tables

This documents the database shape for 3 tables: `users`, `roles`, `user_roles`. It supports:

- Stores application login accounts and audit metadata
- Stores application security roles
- Maps users to granted roles

Implemented by Flyway migration:
`api/src/main/resources/db/migration/V1__create_user_auth_tables.sql`.

## Tables

### `users`

Stores application login accounts and audit metadata.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | `BIGINT` | Yes | Identity primary key |
| `email` | `VARCHAR(255)` | Yes | Unique |
| `password_hash` | `VARCHAR(255)` | Yes |  |
| `enabled` | `BOOLEAN` | Yes | Defaults to `TRUE` |
| `created_at` | `TIMESTAMP` | No | Defaults to `CURRENT_TIMESTAMP`; Creation timestamp |
| `last_edited_at` | `TIMESTAMP` | No |  |
| `last_edited_by_user_id` | `BIGINT` | No | References `users(id)` |

### `roles`

Stores application security roles.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `id` | `BIGINT` | Yes | Identity primary key |
| `name` | `VARCHAR(100)` | Yes | Unique |
| `created_at` | `TIMESTAMP` | No | Defaults to `CURRENT_TIMESTAMP`; Creation timestamp |
| `last_edited_at` | `TIMESTAMP` | No |  |
| `last_edited_by_user_id` | `BIGINT` | No | References `users(id)` |

### `user_roles`

Maps users to granted roles.

| Column | Type | Required | Notes |
| --- | --- | --- | --- |
| `user_id` | `BIGINT` | Yes | Primary key; References `users(id)` |
| `role_id` | `BIGINT` | Yes | Primary key; References `roles(id)` |

## SQL

```sql
CREATE FUNCTION set_updated_at_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE FUNCTION set_last_updated_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    NEW.last_updated = CURRENT_TIMESTAMP;
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TABLE users (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    email VARCHAR(255) NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    enabled BOOLEAN NOT NULL DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_edited_at TIMESTAMP NULL,
    last_edited_by_user_id BIGINT NULL,
    PRIMARY KEY (id),
    CONSTRAINT uk_users_email UNIQUE (email),
    CONSTRAINT fk_users_last_edited_by
        FOREIGN KEY (last_edited_by_user_id) REFERENCES users(id)
        ON DELETE SET NULL
);

CREATE TABLE roles (
    id BIGINT GENERATED BY DEFAULT AS IDENTITY,
    name VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_edited_at TIMESTAMP NULL,
    last_edited_by_user_id BIGINT NULL,
    PRIMARY KEY (id),
    CONSTRAINT uk_roles_name UNIQUE (name),
    CONSTRAINT fk_roles_last_edited_by
        FOREIGN KEY (last_edited_by_user_id) REFERENCES users(id)
        ON DELETE SET NULL
);

CREATE TABLE user_roles (
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    PRIMARY KEY (user_id, role_id),
    CONSTRAINT fk_user_roles_user
        FOREIGN KEY (user_id) REFERENCES users(id)
        ON DELETE CASCADE,
    CONSTRAINT fk_user_roles_role
        FOREIGN KEY (role_id) REFERENCES roles(id)
        ON DELETE CASCADE
);

INSERT INTO roles (name)
VALUES
    ('ADMIN'),
    ('STUDENT'),
    ('FACULTY'),
    ('DEPARTMENT_HEAD'),
    ('ADJUNCT'),
    ('TEACHING_ASSISTANT')
ON CONFLICT (name) DO NOTHING;
```
